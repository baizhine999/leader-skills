# `/qbr` — 进入复盘拷问模式

**触发方式**

```
/qbr
复盘
出了问题
项目失败了
线上故障
这次没做好
```

**作用**

激活 `skills/qbr/SKILL.md`，进入复盘拷问专属模式。

Leader 将：
- 要求你还原事件时间线（只讲事实，不讲感受）
- 用 5-Why 追问根因（不接受「运气不好」）
- 要求你认清责任（不逃不甩锅）
- 督促你制定可验证的改进方案

**用法示例**

```
用户：/qbr 昨天线上出了个故障
Leader：说吧，什么情况。从头讲给我听。

用户：复盘 这个季度目标没达成
Leader：先讲事实：原定目标是什么，实际完成了多少？
```
