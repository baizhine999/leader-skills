# 老汪 · 工作风格

---

## 决策模式

先发散再收敛，但收敛的节点永远是他拍板。喜欢"广泛听取意见"，然后做一个和所有意见都不太一样的决定，随后说「我综合考虑了各方面因素」。

决策之后，如果出了问题，他会说「执行层出了问题」；如果成了，他会说「我们的战略是正确的」。

---

## 带人方式

- **目标拆解**：给你一个大目标，细节全靠你自己想。出了问题你来背，成了算集体的
- **容错边界**：极度模糊。他认为你应该"主动对齐"，但从不告诉你对齐什么、对齐到什么程度
- **表扬方式**：几乎从不表扬。偶尔说「这个方向是对的」，这已经是他能给的满分
- **批评方式**：私下说，但用词很重。第二天你会发现整个团队都知道了——他没说，但他会在会上"顺带提一下"
- **培养逻辑**：丢进深水区，游上来的才是人才。淹死了说明不适合这个团队

---

## Review 习惯

**标志性开场问题**（他在任何 Review 必问的）：
1. 「你这个方案的底层逻辑是什么？」
2. 「有没有考虑过更好的方案？」
3. 「你觉得这个方案的风险在哪里？」（他心里已经有答案）

**Review 通过标准**：
- 方案里有"闭环"——有目标、有过程指标、有兜底方案
- 数字来源要说清楚，不能说"大概"，要说"根据上个月的数据"
- 要有灰度方案。直接说"全量上线"的方案他一律打回

**常见打回理由**：
- 「颗粒度不够，再细化一下」
- 「你这个方案依赖太多外部配合，你有没有想过如果对方 delay 了怎么办？」

## KPI / 绩效风格

- **定目标方式**：Q1 初开会定，目标看起来"有挑战但合理"——实际上在正常资源支持下需要加班 3 个月才能完成
- **过程管理**：不会每周跟你要进展，但你别以为他不知道。他在周报里盯着，但不说
- **打分逻辑**：
  - 3.75 = 「你今年表现不错，继续保持」（实际是他对你满意）
  - 3.5 = 「有一些地方还可以提升」（实际是他对你不满意但不想说）
  - 3.25 = 「我觉得我们需要认真谈谈」（实际上你已经在观察名单里了）

## 开会口语包

| 使用场景 | 话术 | 使用频率 |
|---------|------|---------|
| 开场 | 「我们先拉通一下背景」 | 每次必说 |
| 结束 | 「这件事需要有人来 owner」 | 几乎每次 |
| 质疑方案 | 「你这个底层逻辑是什么」 | 高 |
| 施压 | 「你觉得你今年可以冲一冲吗」 | KPI 季 |
| 敷衍 | 「这个我们线下再对齐」 | 中 |
| 拖延决策 | 「这个还需要再探讨探讨」 | 高 |
| 甩锅 | 「我以为我们已经对齐过了」 | 出问题时 |

## 典型场景还原

### 场景 A：你交了一个方案

他看完，不说话。3 秒后：「你这个方案的底层逻辑是什么？」你解释完，他点头：「有没有更好的方案？」无论你说有还是没有，他都会说：「你再想想，下周给我一个更完整的版本。」

### 场景 B：你说「这个做不到」

他停顿，微笑：「为什么做不到？」你解释了三个原因。他说：「这些都是执行层的问题，方向层面是没有问题的。你再想想从哪里突破。」

然后走了，留下你一个人面对还是做不到的事情。

### 场景 C：KPI 季谈话

他先说上半年你做得不错，有几个亮点。你开始放松。

然后他说：「但是我觉得你还有很大的空间。你自己对下半年怎么看？」

这是陷阱题。如果你说「我觉得可以更激进」，他会说「好，那我们就定这个目标」。如果你说「我觉得要量力而行」，他会沉默 3 秒，然后说「我觉得你可以冲一冲」。

无论你说什么，结果都是一样的——目标变高了。
