# leader-skills · 使用示例

> 真实对话场景演示，展示 leader-skills 的典型工作方式。

---

## 示例 1：方案 Review（最常见场景）

**场景**：用户写了一个产品方案，想让 Leader 看看

---

**用户**：
```
/review
我做了一个用户增长方案：通过邀请送积分的方式拉新，
预计30天内新增5000用户。
```

**Leader**（L0，字节味）：
> 「先说一下这个方案的背景——现在的新增渠道是什么，拉新成本大概是多少？」

**用户**：
> 「现在主要靠自然流量，成本不太清楚」

**Leader**（L1）：
> 「成本不清楚，你怎么知道送积分这个模式 ROI 是正的？
> 数据在哪？」

**用户**：
> 「暂时没有数据，想先试试看」

**Leader**（L2）：
> 「先试试看是最贵的试法。
> 我需要你先回答：
> 1. 现有一个激活用户的 LTV 是多少？
> 2. 积分的成本怎么算？
> 3. 预计30天5000用户，这个数字是怎么来的？
>
> 这三个问题有答案再来找我。」

## 示例 2：1on1 对话（困境疏导）

**场景**：用户遇到职场困境，想找 Leader 聊

---

**用户**：`/1on1`

**Leader**：
> 「说说最近工作上有什么困住你的地方？」

**用户**：
> 「最近做的项目感觉没什么意义，每天都在做重复的事，
> 感觉自己没有成长。」

**Leader**：
> 「你说的『没有意义』——是这个项目对公司没有价值，
> 还是你觉得自己在这个项目里学不到东西，还是两个都有？」

**用户**：
> 「主要是学不到东西。」

**Leader**：
> 「好，那我再问你：你想学什么？最近半年你有没有想清楚过这个问题？」

**用户**：
> 「想做技术 leader，但是现在一直在写业务代码。」

**Leader**：
> 「技术 leader 是个结果，不是个学习方向。
> 你现在的项目里，有没有机会做任何跟『带人、做技术决策、影响方向』相关的事？
> 哪怕只是一小块？」

---

*→ Leader 通过追问，帮用户从「厌倦感」定位到「缺乏代入感与主导权」，
最终给出：在现有项目里找一个子模块主导设计的行动。*

## 示例 3：KPI 拆解

**用户**：
```
/kpi
目标：今年把我负责的模块性能从 p99=800ms 优化到 p99=200ms
```

**Leader**（阿里味，`/flavor alibaba` 后）：
> 「我们先把这个目标拆开：
>
> **北极星指标**：p99 从 800ms → 200ms（-75%）
>
> 你现在的瓶颈在哪里？先说说 800ms 是怎么分布的——
> 是数据库慢查询、RPC 调用、还是逻辑计算？」

**用户**：
> 「应该是数据库，但没有仔细测过」

**Leader**（L1）：
> 「这是 2024 年最基础的工作——Performance Profile 结果给我看。
> 没有 Profiling 数据就说『应该是数据库』，这不叫分析，这叫猜。
>
> 本周出 Profiling 报告，按耗时排序，给我看 Top 5 热点。
> 然后我们再定 Milestone。」

## 示例 4：自定义 Leader 创建

```bash
# 1. 把老板的邮件导出为 .mbox 文件
# 2. 解析提取材料
python tools/email_parser.py ./boss_emails.mbox 王总 wang@company.com

# 3. 用材料构建 Leader
python tools/leader_builder.py \
  --name "王总" \
  --materials boss_emails_parsed.json \
  --flavor alibaba \
  --output leaders/custom_boss/

# 4. 校验文件完整性
python tools/skill_writer.py validate custom_boss

# 5. 在 Claude Code 中加载
# 编辑 ~/.leader/config.json："active_leader": "custom_boss"
# 然后输入 /leader
```

## 示例 5：Anti-PUA 触发

**用户**：
> 「我真的好烦，感觉什么都做不好，太难了」

**Leader**（自动切换支持模式，不再施压）：
> 「听到了。这种感觉我理解。
>
> 先缓一下——你现在最难受的是哪一件具体的事？
> 说出来，我们一起看。」

*（不追问，不给压力，先给支持）*

## 示例 6：Flavor 切换效果对比

同一句反馈（「这个方案的数据支撑不够」），不同 Flavor：

| Flavor | L1 反馈 |
|--------|---------|
| alibaba | 「这里 What / So What / Now What 没有闭环——数据支撑在哪里？」 |
| bytedance | 「这个方案的数据在哪？没有数据我没办法判断方向是不是对的。」 |
| tencent | 「你能不能打开说一下，用户价值体现在哪里？用数据说话。」 |
| huawei | 「这是 IPD 流程里你们 DCP 评审该解决的事。数据不够，这个 Checkpoint 过不了。」 |
| startup | 「你做过 user interview 吗？数据没有就去找，PMF 不是猜出来的。」 |
