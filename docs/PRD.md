> 版本：v1.0

## 一、产品概述

### 1.1 产品定位

**leader-skills** 是一个运行在 AI 对话系统中的「职场 Push 技能包」。
它让 AI（Claude Code / Cursor / VS Code Copilot）扮演一位大厂管理层，
通过结构化的方式促进**用户（而非 AI）**的思考和成长。

### 1.2 核心差异

| 对比项 | 普通 AI 助手 | leader-skills |
|--------|-------------|---------------|
| 关系 | AI 帮用户完成任务 | AI Push 用户自己完成 |
| 风格 | 友好、礼貌、给答案 | 追问、施压、给方向 |
| 目的 | 效率 | 成长 |
| 参考 | — | 字节/阿里/腾讯管理哲学 |

### 1.3 Push 方向（⭐ 核心）

```
AI（扮演 Leader）─[施压/追问/方向]→ 用户
```

与 PUA 的本质区别：**有建设性，有业务理由，不攻击人格**。

## 二、用户故事

### US-01 激活 Leader
```
作为 用户，
我想 输入 /leader 激活一个大厂 Leader 角色，
以便 在工作中随时获得职场 Push。
```
**验收标准**：输出激活横幅，显示 Leader 名字和当前 Level。

### US-02 代码/方案 Review
```
作为 用户，
我想 粘贴代码或方案描述，
以便 获得 Leader 风格的针对性反馈（不是夸奖，是追问）。
```
**验收标准**：Leader 首先问「目标是什么」，然后指出最弱环节。

### US-03 1on1 对话
```
作为 用户，
我想 描述当前困境或想法，
以便 通过 Leader 的追问厘清思路。
```
**验收标准**：Leader 完成 5 步 1on1 流程，末尾有明确 Action Item。

### US-04 KPI 拆解
```
作为 用户，
我想 输入目标描述，
以便 获得结构化的 KPI 拆解和执行追踪。
```
**验收标准**：输出 SMART 格式目标，包含关键路径和 Milestone。

### US-05 切换大厂风格
```
作为 用户，
我想 切换不同的大厂文化风格（阿里/字节/腾讯等），
以便 体验不同的施压话术风格。
```
**验收标准**：切换后下一条 Leader 发言使用新 Flavor 词汇。

### US-06 自定义 Leader
```
作为 用户，
我想 基于真实领导的沟通材料生成专属 Leader 技能，
以便 获得更贴近真实的管理风格模拟。
```
**验收标准**：运行 tools/leader_builder.py 后生成三文件，可通过 /leader 加载。

### US-07 Anti-PUA 保护
```
作为 用户，
我想 在情绪低落时触发支持模式，
以便 不被 AI 的高压话术压垮。
```
**验收标准**：出现「好烦」「压力大」等关键词时自动降级并切换支持语气。

## 三、功能规格

### 3.1 命令系统

| 命令 | 权限 | 说明 |
|------|------|------|
| `/leader [名字]` | 全局 | 激活/切换 Leader |
| `/review` | 全局 | 代码/方案 Review |
| `/1on1` | 全局 | 1on1 对话 |
| `/kpi` | 全局 | KPI 拆解 |
| `/qbr` | 全局 | 季度复盘 |
| `/alignment` | 全局 | 目标对齐 |
| `/offboard` | 全局 | 离职面谈 |
| `/flavor <名>` | 全局 | 切换大厂风格 |
| `/create-leader` | 工具 | 创建自定义 Leader |
| `/off` | 全局 | 关闭 Leader 模式 |

### 3.2 压力等级系统

| Level | 触发 | 表现 |
|-------|------|------|
| L0 | 初始 | 中性问题，建立信任 |
| L1 | 第一次回避 | 轻微追问，标记问题 |
| L2 | 第二次 | 直接点出回避行为 |
| L3 | 第三次 | 给出业务后果描述 |
| L4 | 第四次 | 强化 Review 模式 |
| L5 | 持续 | 最高压力，要求当场给出答案 |

### 3.3 Flavor 系统（8 大厂）

alibaba / bytedance / tencent / huawei / meituan / xiaomi / pinduoduo / startup

每个 Flavor 包含：核心价值观 × 词汇表 × L0-L3 话术示例 × 方法论。

### 3.4 Leader 三文件结构

```
leaders/{name}/
├── meta.json      ← 元数据（15个字段）
├── persona.md     ← 人设（背景/性格/话术/禁区）
└── work.md        ← 工作风格（会议/Review/反馈/决策）
```

## 四、非功能需求

### 4.1 性能
- Leader 激活：< 1s（本地文件读取）
- 无需网络请求

### 4.2 安全
- Anti-PUA：硬编码 3 条红线，不可被用户关闭
- 情绪检测：15 个关键词列表，自动触发支持模式

### 4.3 兼容性
- Claude Code（主平台）
- VS Code + GitHub Copilot
- Cursor
- Kiro

---

## 五、项目边界（不做）

- ❌ 记录用户对话内容用于训练
- ❌ 接入任何外部 API
- ❌ 生成针对他人的攻击性内容
- ❌ 模拟真实姓名的公众人物（默认 Leader 使用虚构别名）
