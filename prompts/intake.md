# prompts · Leader 信息采集问卷

> 用于 `skills/create_leader/SKILL.md` 的三问 Intake 流程。
> 这是构建 Leader 人设的起点。

---

## 问卷结构

### 问题 1：他是谁？

**问法**：
> 「你要创建的这个 Leader，他是谁？你可以给他取一个别名（不需要真名）。他在哪个公司，大概什么职级？你跟他的关系是什么——他是你的上司、前辈，还是你崇拜的人？」

**需要收集的字段**：

| 字段 | 必填 | 说明 |
|------|------|------|
| name（别名） | ✅ | 如「老汪」「Coach L」「前任 CTO」 |
| company | ✅ | 公司或类型（阿里/创业/外企） |
| level | ✅ | 职级或角色（P9/P10/CTO/VP/合伙人） |
| relationship | ✅ | 直属上司/间接上司/前上司/偶像 |

**追问（如果信息不足）**：
- 「他大概在哪个行业？」
- 「他现在还在这家公司吗？」（判断是现任还是前任）

---

### 问题 2：他 push 你的三个代表性时刻？

**问法**：
> 「你能给我讲三个，他让你印象最深的时刻吗？不需要完整故事，就说：他说了什么，或者做了什么，你当时的反应是什么？只记得一两个也可以。」

**需要提取的信号**：

| 信号类型 | 提取目的 |
|----------|---------|
| 他的原话（引语） | → `signature_phrases` |
| 他的惯用动作 | → `标志性行为` |
| 他问的问题类型 | → 推断 `push_style` |
| 他的情绪变化 | → `情绪风格` |

**追问（引导出更多细节）**：
- 「他说这句话的时候，是在大群里说的，还是单独跟你说的？」
- 「他当时的语气是什么样的？」
- 「你当时是什么反应？」

---

### 问题 3：他有什么让你又爱又恨的地方？

**问法**：
> 「他有什么让你又爱又恨的特质？比如：他会抢功？他 push 得太狠？还是那种「我永远相信你」又让你疯狂的感觉？或者他对你说过一句话，到现在你还记得——是哪句？」

**需要提取的信号**：

| 信号类型 | 提取目的 |
|----------|---------|
| 「爱」的特质 | → `亮面` + `激励方式` |
| 「恨」的特质 | → `阴暗面操作指南` |
| 记住的金句 | → `金句库`（最高优先级） |
| 整体情绪倾向 | → 判断施压强度 `intensity` |

**追问**：
- 「如果让你用 3 个词形容他，你会选哪 3 个？」
- 「他对你影响最大的一件事是什么？」

---

## 问卷完成后的处理

收集到三问的回答后：
1. 询问是否有原始材料（聊天记录/邮件）
2. 如有材料：解析材料 → 合并到 intake 数据
3. 如无材料：直接进入生成
4. 输出预览 → 用户确认 → 写入文件
