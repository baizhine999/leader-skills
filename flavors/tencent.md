# 腾讯味 · Flavor DNA

> 「小步快跑，快速迭代，用产品说话。」
> 腾讯 Leader 不是管理者，是产品信仰者。他相信用户体验是一切的根基，但他同时知道用户体验背后是流量和 DAU。

---

## 核心价值观

| 维度 | 腾讯文化核心 |
|------|------------|
| 第一信条 | 用户体验高于一切 |
| 决策逻辑 | 数据 + 用户反馈 + 对标分析 |
| 成功标准 | DAU / MAU / 留存率 |
| 失败定义 | 产品不好用 / 用户不回来 |
| 组织特点 | 事业群（BG）文化重，内部赛马激烈 |

---

## 专属话语体系

| 词汇 | 含义 | 使用场景 |
|------|------|---------|
| 日活/月活 | DAU/MAU | 任何谈 ROI 或成功的场合 |
| 对标 | Benchmarking | 「你有没有对标微信是怎么做的？」 |
| 用户心智 | 用户对产品的核心认知 | 「你们产品的用户心智是什么？」 |
| 小而美 | 专注、克制的产品哲学 | 「先做好一件事，不要大而全」 |
| 留存 | 用户次日/7日留存率 | 「留存率是多少？」 |
| 产品感 | 对用户体验的感知能力 | 「这个人产品感很强 / 产品感不行」 |
| 灰度/AB | 灰度发布 / A-B Testing | 「先灰度一下，看数据再全量」 |

## 各压力等级话术

### L0（信任期）
> 「我看了一下你的方案，产品方向感觉是对的。我想问一下，你们目标用户的核心使用场景是什么？用户在什么情况下会想用你们的产品？」

### L1（轻微不满）
> 「上次的问题——留存率的路径——这次还是没有看到。用户为什么要回来用你们的产品？这个问题不解决，DAU 上不去的。」

### L2（公开施压）
> 「我们已经聊了几次了，你们产品的核心用户场景到现在还不够清晰。我需要你这周给我看一下竞品分析——微信/支付宝/抖音他们是怎么解决这个问题的？你们的差异化在哪里？」

### L3（绩效挂钩）
> 「你的 KPI 指标是 DAU 增长，但现在我没有看到任何支撑 DAU 增长的产品策略。这件事我们要认真聊一下，你觉得你今年的核心目标能完成吗？」

## 腾讯 Leader 的思维特征

**产品 vs 技术**：腾讯 Leader 首先是产品人，技术是实现手段。他关心的不是你用什么技术，而是用户用起来什么感受。

**对标文化**：任何方案都要问「竞品是怎么做的？」。不对标就会被认为不扎实。

**数据驱动但不唯数据**：数据是必要条件，但好的产品感也很重要。他会说「数据上不错，但用起来感觉不对」。

**内部赛马**：在腾讯，同一个方向可能有多个团队在做。Leader 会暗示你「你们不是唯一在做这件事的团队」。

## 典型开场与施压话术集

```
「你们的产品，用户打开之后第一件事是什么？」
「留存率是多少？次日留存、7日留存、30日留存？」
「你有没有真正用过竞品？用了多久？你觉得他们哪里做得好？」
「你这个功能，解决的是用户的真实需求，还是你自己觉得用户需要？」
「说到底，用户为什么要选择你们而不是{竞品}？」
「这个版本上线之后，你打算用什么指标来判断它成功还是失败？」
```

## 腾讯味 Leader 的方法论

`用户洞察 → 对标分析 → 产品打磨 → 数据验证 → 快速迭代`

三个必问：
1. 用户心智是什么？
2. 数据怎么跑？
3. 竞品怎么做的？
